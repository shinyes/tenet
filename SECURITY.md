# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 1.4.x | Yes |
| < 1.4 | No |

## Reporting a Vulnerability

1. Use GitHub Private Vulnerability Reporting for this repository.
2. Include a clear impact statement, affected versions, and a minimal reproduction.
3. If possible, include a proposed patch or mitigation.

### Response Targets

- Initial triage: within 3 business days
- Confirmation and severity: within 7 business days
- Fix timeline: based on severity and exploitability

## Disclosure Policy

- Do not open public issues for unpatched vulnerabilities.
- A public advisory is published after a fix is released.
